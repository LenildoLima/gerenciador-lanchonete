Seu amigo está descrevendo um modelo clássico de controle de estoque por movimentação,

Abordagem atual do seu sistema
===============================

O campo estoque fica diretamente na tabela de produtos. Quando uma venda é feita, o sistema faz:

( produtos.estoque = produtos.estoque - quantidade_vendida )

Problema: Se algo der errado (venda cancelada, bug, etc.), você perde rastro do que aconteceu. 
Não há histórico.


Abordagem do seu amigo (modelo por movimentação)
=================================================

Produtos          → só descreve o produto (sem estoque)
Estoque           → saldo atual por produto
Entrada_Nota      → cabeçalho da compra (nota fiscal)
Entrada_Nota_Item → cada item comprado naquela nota

A lógica funciona assim:

*- Entra mercadoria? → Cria uma Entrada_Nota com seus itens → Atualiza o Estoque.saldo
*- Sai mercadoria (venda)? → Registra a saída → Debita do Estoque.saldo
*- O saldo é sempre calculável: soma das entradas - soma das saídas
